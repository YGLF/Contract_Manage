# 支持矩阵与失败信号说明

## 1. 环境支持矩阵

| 环境 | 角色 | 说明 |
|---|---|---|
| 麒麟 V10 SP3 | 生产基线 | 正式生产环境，优先使用 `docker compose` |
| CentOS 7 | 部署测试环境 | 仅用于应用系统部署测试，不作为生产基线 |
| 本机无 Docker | 仅开发文档查看 | 不支持执行容器化部署与健康检查 |

## 2. Compose 命令约定

- 优先使用 `docker compose`
- 现场如果只有 `docker-compose`，脚本会自动回退
- 生产交付仍建议统一安装和使用受控的 Compose 方案

## 3. 失败信号说明

- `FAIL`：必须先修复，不能继续部署
- `WARN`：可以继续，但需要人工确认风险
- `unhealthy`：按失败处理，不可视为健康
- `Exited` / `Restarting`：按失败处理

## 4. 常见排障顺序

1. 先看 `scripts/check-env.sh` 输出
2. 再看 `scripts/health-check.sh` 输出
3. 再看对应服务日志
4. 再确认数据库、910B、下游 URL、权限和端口

## 5. 适用边界

本说明只覆盖当前微服务交付版的部署和运维边界，不改变业务接口、审批流程或审计语义。
