# 合同管理系统微服务离线部署手册

## 1. 适用场景

本手册用于以下场景：

- 目标服务器在政企内网，不能直接访问公网
- 目标服务器不能在线拉取 Docker 镜像
- 目标服务器不能在线下载 Go 依赖

当前项目有两条部署路径：

1. `docker-compose.microservices.yml`：开发/联调路径，容器内 `go run`，需要 `golang:1.21` 和 `vendor/`
2. `docker-compose.production.yml`：生产镜像路径，在联网机器先构建并导出每个微服务镜像，内网服务器只导入镜像并运行预编译二进制

CentOS 7 仅作为应用系统部署测试环境，生产基线仍是麒麟 V10 SP3。

生产部署优先使用 `docker-compose.production.yml`。开发/联调路径的离线风险点有两个：

1. `docker-compose.microservices.yml` 依赖 `golang:1.21` 镜像
2. `docker compose` 启动时各服务通过 `go run` 运行，如果没有 `vendor/`，Go 可能尝试在线拉依赖

所以离线部署必须先准备：

- 本地 Docker 镜像，包括 `anxin-contract/*` 业务镜像
- 本地 `vendor/` 依赖目录

## 2. 联网机器上先做什么

在一台可以联网的 Linux 机器上，进入项目根目录执行：

```bash
bash scripts/prepare-offline-bundle.sh
```

这个脚本会做 4 件事：

1. 执行 `go mod vendor`
2. 拉取并导出 `golang:1.21`
3. 通过 `docker-compose.production.yml` 构建所有 `anxin-contract/*` 业务镜像
4. 导出生产业务镜像 tar 包
5. 生成 `offline-bundle/`
6. 生成一个离线交付压缩包

执行完成后，重点确认这两个内容存在：

- `offline-bundle/vendor/`
- `offline-bundle/images/golang_1.21.tar`
- `offline-bundle/images/anxin_contract_services_1.1.tar`

## 3. 如何把离线包带到内网服务器

将以下内容复制到内网服务器：

1. 整个项目目录
2. `offline-bundle/` 目录

建议目录形态如下：

```text
AnXin_Contract_Manage_microservices_v1.1/
├── cmd/
├── docs/
├── internal/
├── pkg/
├── scripts/
├── docker-compose.microservices.yml
├── .env.microservices.example
└── offline-bundle/
    ├── images/
    ├── vendor/
    └── README.txt
```

## 4. 内网服务器如何补齐环境

进入项目根目录后执行：

```bash
bash scripts/bootstrap-runtime.sh ./offline-bundle
```

这个脚本会做：

1. 检查 Docker、Docker Compose、curl、tar
2. 自动创建 `uploads/`
3. 如果 `.env` 不存在，就从模板生成
4. 将 `offline-bundle/vendor/` 同步到项目 `vendor/`
5. 导入离线镜像 tar 包
6. 检查 `anxin-contract/gateway-service:${APP_VERSION:-1.1}` 生产网关镜像是否已经可用
7. 如果存在 `golang:1.21`，同时提示开发/联调路径可用

## 5. 环境补齐后如何启动

先编辑 `.env`，至少改这几项：

```env
JWT_SECRET=replace-with-strong-secret
DB_HOST=10.0.0.10
DB_PORT=3306
DB_USER=contract_user
DB_PASSWORD=replace-password
DB_NAME=contract_manage
AI_MODEL_ENDPOINT=http://910b-inference.internal:9000/infer
UPLOAD_DIR=uploads
```

其中 `DB_USER` 和 `DB_PASSWORD` 必须来自现场 DBA 或密钥管理流程。`init.sql` 不提供默认数据库账号，也不执行全权限授权；离线部署前应确认业务数据库已创建、账号已按最小权限授权，并且 `scripts/check-env.sh` 可以连通数据库端口。

生产镜像路径按顺序执行：

```bash
COMPOSE_FILE=./docker-compose.production.yml bash scripts/check-env.sh
docker compose --env-file .env -f docker-compose.production.yml up -d
COMPOSE_FILE=./docker-compose.production.yml bash scripts/health-check.sh
```

开发/联调路径仅用于源码级联调：

```bash
bash scripts/check-env.sh
bash scripts/start-microservices.sh
bash scripts/health-check.sh
```

生产镜像路径不挂载源码，也不在容器启动时执行 `go run`。目标环境完全离线时，不应在内网服务器执行镜像构建；应使用联网机器准备好的 `anxin_contract_services_*.tar` 导入后直接启动。

## 6. 内网部署常见失败点

### 6.1 `pull access denied` 或业务镜像不存在

原因：

- 服务器无公网
- 本地没有导入 `anxin-contract/*` 生产业务镜像

处理：

```bash
docker images | grep anxin-contract
bash scripts/bootstrap-runtime.sh ./offline-bundle
```

如果使用开发/联调路径，再检查 `golang:1.21`：

```bash
docker images | grep golang
```

### 6.2 `go: module lookup disabled` / `cannot download module`

原因：

- 没有 `vendor/`
- 离线包未同步完整

处理：

1. 检查 `offline-bundle/vendor/` 是否存在
2. 重新执行：

```bash
bash scripts/bootstrap-runtime.sh ./offline-bundle
```

### 6.3 `connection refused`

原因通常是：

- GBADE 没通
- 910B 没通
- 某个内部服务没起来

处理：

```bash
bash scripts/check-env.sh
docker compose -f docker-compose.microservices.yml ps
docker compose -f docker-compose.microservices.yml logs --tail=200
```

## 7. 当前方案边界

当前这套离线方案已经覆盖：

- 离线镜像导入
- 生产业务镜像预构建和导出
- 离线 Go 依赖补齐
- 内网服务器环境补齐
- 启动前检查和健康检查

但仍有两个边界要明确：

1. 目标服务器必须先安装 Docker 和 Docker Compose，本项目不负责离线安装 Docker 本体
2. `docker-compose.microservices.yml` 仍是源码挂载 + `go run` 方式，仅建议用于联调；生产优先使用 `docker-compose.production.yml`
