# 安信合同管理系统 - 监控告警配置

本文档提供系统监控和告警配置指南。

## 1. 监控架构

### 1.1 监控目标

- 及时发现系统异常
- 定位故障原因
- 评估系统性能
- 预防潜在问题

### 1.2 监控指标

| 类别 | 指标 | 阈值 |
|------|------|------|
| 系统 | CPU使用率 | > 80% |
| 系统 | 内存使用率 | > 85% |
| 系统 | 磁盘使用率 | > 80% |
| 应用 | API响应时间 | > 2s |
| 应用 | 错误率 | > 1% |
| 数据库 | 连接数 | > 80% |
| 数据库 | 慢查询 | > 1s |

## 2. 系统监控

### 2.1 基础监控

使用 node_exporter + Prometheus + Grafana：

```yaml
# docker-compose.monitoring.yml
version: '3.8'
services:
  prometheus:
    image: prom/prometheus
    ports:
      - "9090:9090"
    volumes:
      - ./prometheus.yml:/etc/prometheus/prometheus.yml
  
  grafana:
    image: grafana/grafana
    ports:
      - "3001:3000"
```

### 2.2 告警规则

```yaml
# prometheus.yml
groups:
- name: contract-manage
  rules:
  - alert: HighCPUUsage
    expr: cpu_usage > 80
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "CPU使用率过高"
      
  - alert: HighMemoryUsage
    expr: memory_usage > 85
    for: 5m
    labels:
      severity: warning
    annotations:
      summary: "内存使用率过高"
      
  - alert: DiskSpaceLow
    expr: disk_free < 20
    labels:
      severity: critical
    annotations:
      summary: "磁盘空间不足"
```

## 3. 应用监控

### 3.1 日志监控

生产微服务运行时建议同时纳入以下基础存活探针监控：

- `gateway-service`
- `identity-service`
- `audit-service`
- `contract-service`
- `document-service`
- `approval-workflow-service`
- `notification-service`
- `report-service`
- `search-ai-service`

这些服务在生产 compose 中统一暴露 `/health` 容器级探针，可用于基础 liveness 监控；但它们不等价于数据库、下游微服务或 910B 推理接口的 readiness，应额外配置数据库连通、下游 5xx 比例、AI 调用失败率等告警。

如果容器状态出现 `unhealthy`，应单独作为告警条件处理，不要只依赖 `Exited` 或 `Restarting`。

使用ELK Stack：

```yaml
# filebeat配置
filebeat.inputs:
- type: log
  paths:
    - /var/log/contract-manage/*.log
  fields:
    service: contract-manage
```

### 3.2 链路追踪

使用 Jaeger：

```go
// 初始化 tracer
import "github.com/uber/jaeger-client-go"

config := jaeger.Configuration{
    ServiceName: "contract-manage",
    Sampler:     &jaeger.SamplerConfig{Type: jaeger.SamplerConstRate, Param: 1},
}
```

## 4. 数据库监控

### 4.1 慢查询监控

```sql
-- 启用慢查询日志
SET GLOBAL slow_query_log = 'ON';
SET GLOBAL long_query_time = 1;
SET GLOBAL log_output = 'FILE';

-- 查看慢查询
SELECT * FROM mysql.slow_log 
ORDER BY start_time DESC LIMIT 100;
```

### 4.2 连接监控

```sql
-- 查看当前连接
SHOW STATUS LIKE 'Threads_connected';
SHOW STATUS LIKE 'Max_used_connections';

-- 查看执行中的查询
SHOW PROCESSLIST;
```

## 6. 监控面板

### 6.1 Grafana面板

| 面板名称 | 展示内容 |
|----------|----------|
| 系统概览 | CPU/内存/磁盘/网络 |
| 应用性能 | QPS/延迟/错误率 |
| 数据库 | 连接数/查询统计 |
| 业务指标 | 合同数量/审批数量 |

### 6.2 关键指标

```
# API响应时间
histogram_quantile(0.95, rate(http_request_duration_seconds_bucket[5m]))

# QPS
rate(http_requests_total[5m])

# 错误率
rate(http_requests_total{status=~"5.."}[5m]) / rate(http_requests_total[5m])
```

## 7. 巡检配置

### 7.1 自动巡检

使用 Prometheus Alertmanager 定时检查：

```yaml
- name: daily-check
  interval: 24h
  rules:
  - alert: ServiceDown
    expr: up{job="contract-manage"} == 0
    for: 1m
    labels:
      severity: critical
```

### 7.2 巡检项目

| 检查项 | 检查内容 | 告警条件 |
|--------|----------|----------|
| 服务状态 | 后端进程 | 不存在 |
| 容器健康 | production compose `/health` 探针 | `unhealthy` |
| 数据库 | 连接状态 | 无法连接 |
| 磁盘空间 | 使用率 | > 80% |
| 内存使用 | 使用率 | > 85% |
| 错误日志 | 最新错误 | 存在ERROR |

## 8. 应急响应

### 8.1 告警响应流程

```
收到告警 → 确认问题 → 初步处理 
    → 升级问题 → 持续监控 → 关闭告警
```

### 8.2 常见告警处理

| 告警 | 处理措施 |
|------|----------|
| CPU过高 | 检查进程/重启服务 |
| 内存不足 | 释放缓存/重启服务 |
| 磁盘满 | 清理日志/扩展磁盘 |
| 服务不可用 | 检查进程/检查日志 |
