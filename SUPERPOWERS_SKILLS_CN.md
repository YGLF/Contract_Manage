# Superpowers Skills 中文备注

| Skill | 中文备注 |
|---|---|
| `brainstorming` | 先做需求澄清和方案设计，再进入实现。 |
| `dispatching-parallel-agents` | 有多个互不依赖的任务时，并行分派给多个子代理执行。 |
| `executing-plans` | 按已经写好的实施计划逐步执行。 |
| `finishing-a-development-branch` | 开发完成后做收尾、合并、PR 或清理。 |
| `receiving-code-review` | 收到代码评审意见后如何判断和处理。 |
| `requesting-code-review` | 在交付前主动发起代码评审。 |
| `subagent-driven-development` | 用子代理拆分任务并推进实现。 |
| `systematic-debugging` | 遇到 bug、失败或异常时，按系统化流程排查。 |
| `test-driven-development` | 先写测试，再写实现，按 TDD 流程开发。 |
| `using-git-worktrees` | 需要隔离开发环境时，使用 git worktree。 |
| `using-superpowers` | 说明如何发现和使用 Superpowers 的总入口技能。 |
| `verification-before-completion` | 完工前先验证结果，再宣布完成。 |
| `writing-plans` | 把需求整理成可执行的实施计划。 |
| `writing-skills` | 创建、修改和验证新的 skills。 |
